**教學單元：**

1. Linux 發行版選擇與系統架構概論

| 第四節 | [實作] 模擬三人團隊協作，完成一輪完整的開發週期 | 17:00 - 18:00 |

**教學單元：**

1. Git 物件模型：blob / tree / commit / tag 的本質
2. 分支策略比較：Git Flow / GitHub Flow / Trunk-Based Development
3. Merge vs Rebase：選擇時機與團隊規範
4. Pull Request 審查流程與 Code Review 最佳實踐
5. 合併衝突的預防與解決策略
6. GitHub Actions 基礎：觸發條件、Job、Step 設計
7. GitLab CI/CD Pipeline 結構與 Runner 設定
8. .gitignore、commit 訊息規範與 tag 管理

**課後作業：** 建立一個 GitHub Repo，設定分支保護規則，用 Git Flow 模式完成至少三個 PR（含衝突解決）

### Week 5 -- 機器學習基礎與 scikit-learn 實戰

機器學習不是只有深度學習。這一週回到最基本的統計學習方法，這些演算法在很多場景下比神經網路更實用、更好解釋。

| 節次 | 內容 | 時間 |
|------|------|------|
| 第一節 | 資料科學工作流程：從原始資料到可訓練資料集 | 14:00 - 15:00 |
| 第二節 | 監督式學習：分類與迴歸的核心演算法 | 15:00 - 16:00 |
| 第三節 | 非監督式學習與模型評估方法論 | 16:00 - 17:00 |
| 第四節 | [實作] 用 scikit-learn 完成端到端 ML Pipeline | 17:00 - 18:00 |

**教學單元：**

1. 探索式資料分析（EDA）：理解你的資料長什麼樣
2. 資料前處理：缺值處理、編碼轉換、標準化與正規化
3. 特徵工程：從原始欄位萃取有預測力的特徵
4. 分類演算法比較：Logistic Regression / SVM / Random Forest / XGBoost

7. 模型評估：交叉驗證、混淆矩陣、AUC-ROC、過擬合診斷
8. scikit-learn Pipeline：串接前處理與模型訓練的自動化流程

**課後作業：** 選擇一個 Kaggle 資料集，完成完整的 ML 專案（EDA 報告 + 特徵工程 + 至少三種演算法比較 + 最佳模型選擇）

### Week 7 -- 自然語言處理與 Transformer

### Week 10 -- RAG 系統設計與建構

LLM 的知識有截止日期，也不懂你公司內部的資料。RAG 讓 LLM 能「查資料再回答」，這是目前企業 AI 應用最重要的架構模式。

| 節次 | 內容 | 時間 |
|------|------|------|

| 第二節 | 向量化與檢索：Embedding 模型、向量資料庫、搜尋策略 | 15:00 - 16:00 |
| 第三節 | LangChain 框架核心：Chain、Retriever、Memory | 16:00 - 17:00 |
| 第四節 | [實作] 用公司文件建構一個 RAG 知識問答系統 | 17:00 - 18:00 |

**教學單元：**

1. RAG vs Fine-tuning：何時該用哪個方案
2. 文件載入與前處理：PDF / Word / Markdown / 網頁擷取
3. 文件切割策略：Chunk Size、Overlap、語意切割的取捨
4. Embedding 模型比較與選擇（OpenAI / Cohere / 開源模型）
5. 向量資料庫實戰：PGVector / ChromaDB / Pinecone

7. LangChain 核心元件串接與自訂 Chain 設計

**課後作業：** 建立一個 RAG 系統，使用至少 20 份文件作為知識庫，並用測試問題集評估檢索品質
