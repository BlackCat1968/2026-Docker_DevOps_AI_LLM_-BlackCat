## 1-2　環境確認：先確定 Docker 真的活著

在裝任何 Python 程式之前，先確認引擎有在動。下面每一條都是可以直接貼進終端機的系統指令。

```bash
docker version
docker info
docker run --rm hello-world
```

逐項詳細解釋：

1. `docker version`：印出用戶端（你打指令的 CLI）和伺服端（背景那顆 Docker 引擎）各自的版本。如果只看到用戶端、伺服端那段噴錯，代表引擎沒開，先去把 Docker Desktop 或 docker 服務啟動起來。
2. `docker info`：印出這台機器目前的整體狀態，例如已經有幾個映像檔、幾個容器、用的是哪種儲存驅動。這條是你日後排查問題的第一站。
3. `docker run --rm hello-world`：跑官方的測試映像檔。它會做完整流程給你看——本機沒有這個映像檔 → 自動從 Docker Hub 下載 → 產生容器 → 印一段歡迎訊息 → 結束。`--rm` 表示「容器一跑完就自動刪掉」，不要留一堆殘骸。

> Tip：`--rm` 是你會用一輩子的旗標。任何「跑一次、看結果、不需要留著」的容器都加上它，能省下你之後手動清理的功夫。
